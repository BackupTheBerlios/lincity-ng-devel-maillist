So correct simplified algorithm should be:
 available_workers = attempt of put_jobs (global number)
 unemployment_1 = failed put_jobs (residence property)
 unemployment_2 = successful_put_jobs - successful_get_jobs (global number)
 lack_of_workers = failed get_jobs (factory property)

 unemployment = unemployment_1 + max(0, unemployment_2)
 people_with_job = available_workers - (unemployment_1 + unemployment_2)
	would be = successful get_jobs if blacksmith and farm didnt mess it.

 available_jobs = people_with_job + lack_of_workers 
	would be = sum of get_jobs (succeeded or failed) if farms ...

1/ For "jobs" in pbar:
- we have to choose what we want to show, people_with_job or available_jobs

2/ For minimap:
- unemployment_1 is current minimap, residence property
- lack_of_worker can be represented as a factory property.
- unemployment_2 would need to be represented too, maybe as a
   market property, to illustrate the trend of jobs in market.

3/ For ECO graph
- if negative unemployment is allowed, then replace the current yellow graph
 by unemployment_1 + unemployment_2
