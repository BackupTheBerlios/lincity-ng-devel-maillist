Hi,

I see several problems with the framedrop changes in r1493. 
One frame per month looks very stuttering.
I would prefer a dynamic framedrop which would only drop as many frames as needed to reach a predefined simulation speed.
Due to the calls to SDL_Delay(10) at every step lincity just uses 30% of CPU on my computer in fast mode.
The help does not reflect the change and has no information about the framedrop or the fact that on slow computers quickness="9" is now faster than quickness="1".
Additionally the frames per second graph does not show the frames/second, but the simulation steps/second. Maybe it could be renamed.
As a long term solution we could decouple the animation from the simulation. Every module could have another function which decides which image to use. This function would be called before every redraw.
