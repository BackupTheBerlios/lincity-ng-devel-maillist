Hi

I believe that the informations (mps_...) about how work a building
are sometimes misleading, and can cause some confusion to the player,
who believe the building does not work because it has 0% in inventory
but at the same time shows efficiency 100%.
An easy workaround would be to replace "inventory" by "surplus". This
is exactly the same thing as the inventory is done after the building
did its production, but seems much more clear if you don't have read
the modules sources.

But this need some changes in all the translations too, so i won't 
commit this change without explicit approval (i'm getting more cautious :-)

Also for light and industries, the translation is
_("light") _("industry") and it should be _("Light industry")
because at least in french the translation is not word by word,
and is in reverse order "industrie légère".

Last, _if_ we want to display more precise information on how/why a
building work or not, we can store information in (x+1,y)... like
what is done for "port" and display these in mps_ . I don't know if
it is a good idea or not.

Cheers.
Alain
