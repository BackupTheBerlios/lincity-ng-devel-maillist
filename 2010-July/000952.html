Alain,

I think the work that my project is dealing with might be useful to the
project. We are using data from a watershed model to serve as a backend (to
be affixed to LinCity-NG on the frontend). Before I explain the entirety of
what I'm working I think a little background information is in order.

A Low Impact Development (LID) is a water resources 'trick' used to help
rainwater get into the ground so that it doesn't stay on roads, stagnate
(i.e. on a parking lot), build (i.e. and cause flooding), etc. These kinds
of things have to do with the sustainability of the watershed...to make
people more aware of this (and what can be done), I would like to add some
parameters to a custom map that will determine the effectiveness of LID
placement.

I want to add checkbox menus (i.e. like the market) to some of the
structures - players will check mark LID's that they would like and the
map's effectiveness values will determine how good of a decision that
actually is. For my little project, I'm going to strip down much of LinCity
for a kind of proof-of-concept-just-for-show-project-toy example. However,
it would be cool to integrate some of this post-project. With the topography
in place, this could get very interesting. I'll be trying to play around
with the map files by hand (unless you have a more efficient option
available offhand)...could you give me some instruction how to use the
keyboard interface for map manipulation (in the meantime)? I use vim so
terminal-text editing shouldn't be a problem.

Kind Regards,
