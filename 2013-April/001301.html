w00t! It turns out that you need the "Reply-To" header to be the email 
address you've subscribed with and not just the "From" header.  Thus, 
when I run "git format-patch", I have to include --header="Reply-To: 
daniel.santos at pobox.com" or add it to my .gitconfig:

[format]
         headers = "Reply-To: Daniel Santos <daniel.santos at pobox.com>"
[sendemail]
     smtpencryption = ssl
     smtpserverport = 465
     smtpserver = smtp.att.yahoo.com
     smtpuser = danielfsantos at att.net
     smtppass = xxxxxxxxxxxxxxxxxxx
     envelopesender = danielfsantos at att.net
     from = Daniel Santos <daniel.santos at pobox.com>

So if anybody else has this problem (when you use a service like 
pobox.com, but some other account to actually send & receive). Most 
listserves only care about the "From" header, but apparently this one 
wants to know the correct Reply-To header (which I should have set anyway).
