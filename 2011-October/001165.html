Raw "git svn" will still require some cleanup, as tags will be
imported as branches and similar stuff. There is a Ruby script that
will clean that up automatically:

https://github.com/nirvdrum/svn2git

Can be used with something like:

svn2git file:///backup/remote/berlios.de/svnrepo/pingus/  --authors
~/projects/pingus/AUTHORS.txt   --no-metadata

I used it for Pingus and Windstille, it works quite well. One thing to
keep in mind with git is that it doesn't do any branch or tag
versioning, so if one deletes an old unmerged branch or tag, then it
is gone for good along with the attached history. For Pingus I solved
that by renaming/converting all unwanted branches and tags into tags
with the prefix archive/, i.e.:

archive/branches/pingus-0.6
archive/tags/before_xml

This moves them out of the way when doing regular work, but preserves
them for historic purpose.

http://grumbel.blogspot.com/2011/08/converting-from-svn-to-git.html
