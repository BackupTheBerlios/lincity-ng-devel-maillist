Hi,
After a svn checkout, gunzip the patch then move it to lincity-ng/trunk/
You have to use -p0 option to leave the paths of the patch unchanged 
like this:
patch -p0 < track-road-bridge-vs1436.patch
After the compilation, I saw that we have a problem with automatic xml 
generation and the file data/gui/build_bridge_yn.xml is overwritten with 
a size of 0.
As a workaround, back-up it _before_ compilation then restore it later.

I'm looking the manage a stack of tiles for dialogue in dragging mode 
but I'm not ready with it. The first time, build one bridge section in 
the water, answer yes and then later you can build all the bridge with 
dragging at the same time.
