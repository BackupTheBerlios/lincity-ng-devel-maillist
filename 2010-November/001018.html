Hi everybody

I created a project on fedorahosted name for a personnal project
linked to lincity-ng, and ended by using it for a *test* for moving
lincity-ng development.

https://fedorahosted.org/TinyEarth/

* It uses git instead of svn 
	** branches and merges are much more easy and robust
	** easy to work offline

* There is a trac instance for managing tickets, patches etc. which
is very convenient (infinitely better than current berlios stuff)
	** roadmap + milestones
	** ticket with comments in the correct order
	** possible to add patches
	** good looking :-)

* As Wolgang said, the trac wiki should not be used, the current wikimedia is
great.

I think this new site should be used for development, and we should keep the
welll known berlios site for the :
	* mailing list
	* wikimedia
	* releases

If we agree to move dev there, i will create a LinCity-NG project there, and 
reimport sources now i know how to do this (or rename my project if it is
possible). I will also open tickets corresponding to confirmed bugs and so on...

Please your comments/advices/ideas are welcome

Cheers
Alain
