On 2013-02-22 Chang Long <karenlong0123 at gmail.com> wrote:
> other translated PO files, but we couldn't figure out how to
> integrate the translations into the game. We wonder how we can
> change the language setting and build and play the game in another
> language. This way, we can also do testing when we finish Chinese
> translation.

If you don't want to change the machine's language settings you
can set the environment variable LINCITY_LANG to override auto
detection. Something like "LINCITY_LANG=zh_CN lincity-ng" should to
the trick. During startup Lincity will print the language it is
using to the terminal.

There is no need to rebuild the code to integrate a new translation.
All you have to to is put the new .po files in data/locale and
data/locale/gui. Add a new folder in data/help for translated helpfile.

The default font does not contain Chinese characters. So like for
Japanese you need an (symlink to) an appropriate font in data/fonts.
We don't ship a Japanese font ATM because all fonts I could find
came with a licence I was unable to fully understand.
