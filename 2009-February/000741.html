Well, as you know I submitted a patch for the race condition calling the
not-thread-safe function IMG_Load_RW of SDL-image (when configured with
--enable-png-shared).  The patch works fine for me, but I began having
terrible performance problems when trying to debug.  Thus, I eventually
opened some bugs with both kdevelop and gdb.  Well now the problems
mysteriously vanished!  I'm suspecting it's something Gentoo AMD64 related. 
So if you run into serious slowdown (gdb taking 98% of CPU), try adding "set
auto-solib-add off" to your ~/.gdbinit.  However, this still leaves issues
with getting libraries loaded to see the symbols, so it may require you to
manually (at the "GDB cmd:" prompt of the gdb view in kdevelop) to load the
symbols for shared libraries.

None the less, if you do run into this, please try to gather as much info
about it as possible (a core file would be nice).  The gdb bug is here:
http://sourceware.org/bugzilla/show_bug.cgi?id=9831 and the kdevelop bug is
here: http://bugs.kde.org/show_bug.cgi?id=183758

So I just need to clean up some final problems (hopefully I can get that done
now that this issue mysteriously stopped) and get my patch submitted to you guys.
